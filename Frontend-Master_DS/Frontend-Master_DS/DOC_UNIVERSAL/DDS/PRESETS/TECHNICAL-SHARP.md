# Preset: Technical Sharp

Precise, high-contrast, developer-oriented aesthetic. Pages feel engineered and credible.

## Character

- precise
- technical
- sharp

## When To Use

- Developer tools and platforms
- API documentation sites
- Infrastructure or DevOps products
- Technical comparison or benchmark pages
- Products where the audience expects precision over warmth
- Data-heavy interfaces that need visual clarity

## Visual Rules

### Typography

- Posture: technical compact
- Headlines: clean sans-serif, medium weight, tight line height
- Body: compact but readable, standard line height
- Code and monospace: first-class citizen — styled and prominent
- Data labels: clear, technical, no frills

### Color

- Accent: electric blue, bright green, cyan, or high-saturation technical tones
- Backgrounds: dark themes preferred (near-black, deep navy, dark gray)
- Contrast: high — strong foreground/background separation
- Gradients: subtle dark gradients, no warm tones
- Syntax highlighting colors as part of the visual identity

### Shape

- Language: sharp
- Radius: minimal (0-6px)
- Cards: sharp corners, visible borders, structured grids
- Buttons: square or slightly rounded, high-contrast states
- Borders: visible and structural, not decorative

### Surface

- Depth: flat to restrained-depth
- Shadows: minimal — borders and color do the work
- Glass effects: allowed sparingly on dark backgrounds
- Surface treatment: matte, precise, no texture

### Motion

- Character: crisp
- Transitions: fast and precise (100-200ms)
- No decorative animations
- State changes are instant or near-instant
- Hover states: clear and immediate, no delays

### Imagery

- Mode: diagrams, code snippets, architecture visuals, terminal screenshots
- System diagrams and flowcharts as primary visual anchors
- Product UI in context (IDE, terminal, dashboard)
- No stock photography, no lifestyle images
- Icons: outline or monoline, technical style

## DS Token Mapping

- `data-visual`: `sleek` or dark variant
- `data-density`: `balanced` to `compact`
- Shape tokens: reduced radius, visible borders
- Shadow tokens: minimal or none
- Motion tokens: shorter durations

## What This Preset Avoids

- Warm, friendly, or playful tone
- Rounded shapes and soft shadows
- Stock photography or lifestyle imagery
- Decorative gradients unrelated to data visualization
- Long marketing copy when a diagram explains better
