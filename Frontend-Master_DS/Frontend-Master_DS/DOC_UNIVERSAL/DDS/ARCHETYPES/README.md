# DDS Archetypes

Site-level design archetypes for reusable public-facing work.

## Purpose

Modes and presets alone are not enough for full websites.

- A **mode** controls composition behavior.
- A **preset** controls aesthetic personality.
- An **archetype** controls the overall website strategy.

Archetypes answer questions such as:

- What kind of site is this overall?
- Where should trust appear first?
- How aggressive or restrained should CTAs be?
- Which pages should feel louder or quieter?
- What visual mistakes make this kind of site look generic?

Use archetypes for:

- multi-page public websites
- reusable web templates
- theme families aimed at a specific market type
- homepage-led builds where page decisions must stay cohesive across the site

Do not use archetypes for:

- single dashboard screens
- utility/admin-only pages
- isolated component design tasks

## Read Policy

For a full website or reusable template build:

1. Read `DDS/README.md`
2. Read `DDS/OPERATING/DESIGN-DECISION-SYSTEM.md`
3. Read `DDS/OPERATING/AI-EXECUTION-FLOW.md`
4. Read the selected archetype from `DDS/ARCHETYPES/`
5. Then read the selected mode from `DDS/MODES/`
6. Then read the selected preset from `DDS/PRESETS/`

Do not load all archetypes. Choose one.

## Relationship To Other DDS Layers

| Layer | Scope | Question |
|-------|-------|----------|
| Archetype | whole site | What kind of website are we building overall? |
| Page type | single page | What kind of page is this? |
| Intent | single page | What is this page trying to do? |
| Mode | page composition | How should this page behave structurally? |
| Preset | page/site feel | What aesthetic personality should it carry? |

Archetypes are the site-level bridge that keeps multiple pages from feeling like unrelated one-off compositions.
