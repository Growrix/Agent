# Product Launch

For announcement pages, launch microsites, waitlists, early-access campaigns, and high-attention release moments.

## Core Goal

Create energy, focus attention, and drive one high-priority action.

## Audience Pattern

- early adopters
- existing audience
- launch traffic from campaigns or social
- curious technical or product-savvy users

## Site Behavior

- fewer sections
- sharper visual contrast
- tighter message hierarchy
- strong hero dominance
- one action path should clearly win

## Page Role Expectations

- Home/Landing: announcement + product hook + action
- Feature pages: supporting proof only
- About: optional and secondary
- Blog: supportive, not the core destination
- Contact: usually not primary

## Composition Tendencies

- dramatic hero okay
- stronger visual transitions okay
- section count should stay controlled
- pacing escalates toward CTA
- proof must be concise and punchy

## Proof Style

Prioritize:

- screenshots
- short demos
- launch metrics
- short testimonials
- logo associations
- sharp benefit framing

## CTA Strategy

- one primary CTA only
- secondary CTA must not compete strongly
- repeated CTA acceptable when energy rises with each section

## Recommended Mode Pairings

- `Bold Marketing`
- `Editorial Narrative` only for story-led launch narratives

## Recommended Preset Pairings

- `High-Energy Launch`
- `Technical Sharp`

## What To Avoid

- turning a launch page into a full corporate website
- too many section types fighting for attention
- weak hero and overlong explanation before the CTA
- safe corporate styling that removes urgency

## Review Questions

- Does the first screen feel like a launch, not a brochure?
- Is there one clearly dominant CTA?
- Does the page stay focused instead of expanding into a generic marketing site?
