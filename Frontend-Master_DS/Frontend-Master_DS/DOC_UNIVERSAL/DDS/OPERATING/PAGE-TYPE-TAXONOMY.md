# Page Type Taxonomy

Every page must be classified before design decisions begin. The page type determines baseline composition expectations.

## Marketing Pages

### Homepage

- Hero: dominant, full-width, must have a primary visual anchor
- Sections: 5-8 typical, with intentional rhythm variation
- Proof: required (testimonials, metrics, logos, or case studies)
- CTA: primary conversion CTA must appear in hero and late-page
- Mode tendency: Bold Marketing
- Anti-pattern: identical stacked card grids with no visual escalation

### Feature Landing Page

- Hero: statement + product proof or split hero
- Sections: 3-5 focused, explaining one feature thoroughly
- Proof: feature-specific (screenshots, demos, comparisons)
- CTA: single focused conversion action
- Mode tendency: Bold Marketing or Editorial Narrative
- Anti-pattern: listing features without showing them working

### Pricing Page

- Hero: minimal, clarity over drama
- Sections: pricing table + FAQ + trust signals
- Proof: value justification (ROI, comparison, guarantee)
- CTA: plan-specific actions, clear upgrade path
- Mode tendency: Dense Data or Clean SaaS
- Anti-pattern: hiding information that affects purchase decisions

### About Page

- Hero: mission statement or team story
- Sections: story, team, values, milestones
- Proof: human-focused (team photos, quotes, history)
- CTA: subtle, relationship-building
- Mode tendency: Editorial Narrative
- Anti-pattern: generic corporate template with stock photos

### Comparison Page

- Hero: clear positioning statement
- Sections: structured comparison tables, feature breakdowns
- Proof: data-driven (benchmarks, side-by-side)
- CTA: choose-us action tied to comparison
- Mode tendency: Dense Data
- Anti-pattern: biased comparison that looks dishonest

## Content Pages

### Blog Index

- Hero: minimal or none
- Sections: categorized article grid with featured content
- Proof: content quality signals (dates, read time, authors)
- CTA: subscribe or explore
- Mode tendency: Editorial Narrative
- Anti-pattern: undifferentiated grid of cards

### Blog Article

- Hero: title + meta, optional cover image
- Sections: content-driven, follows writing structure
- Proof: author authority, related content
- CTA: contextual (next article, subscribe, product link)
- Mode tendency: Editorial Narrative
- Anti-pattern: heavy chrome around simple content

### Case Study

- Hero: headline result + company logo
- Sections: challenge → solution → results
- Proof: metrics, quotes, before/after
- CTA: contact or similar-results action
- Mode tendency: Editorial Narrative
- Anti-pattern: wall of text without visual proof

### Documentation Hub

- Hero: search + navigation
- Sections: categorized doc tree, quick links
- Proof: none required
- CTA: none, utility-focused
- Mode tendency: Clean SaaS
- Anti-pattern: hiding navigation behind too many clicks

## Application Pages

### Dashboard Overview

- Hero: none, data-first
- Sections: metric cards + activity feed + quick actions
- Proof: none
- CTA: contextual actions within data
- Mode tendency: Clean SaaS
- Anti-pattern: vanity metrics without actionable context

### Settings Page

- Hero: none
- Sections: grouped settings with clear labels
- Proof: none
- CTA: save/apply per section
- Mode tendency: Clean SaaS
- Anti-pattern: a single long form with no grouping

### Onboarding Flow

- Hero: progress indicator + current step
- Sections: single focused task per step
- Proof: reassurance (what happens after, skip options)
- CTA: next step, always visible
- Mode tendency: Clean SaaS or Mobile Native
- Anti-pattern: too many fields per step

### Admin Panel

- Hero: none
- Sections: data tables + filters + bulk actions
- Proof: none
- CTA: CRUD actions per row or batch
- Mode tendency: Clean SaaS or Dense Data
- Anti-pattern: displaying everything without filtering or hierarchy

## Classification Rules

1. Classify before designing. Never skip this step.
2. One page = one primary type. Do not hybridize types.
3. If a page does not fit a known type, document the custom classification in the wireframe brief.
4. The page type constrains mode selection but does not force it.
5. Use the listed anti-patterns as a review checklist after composition.
