# Architecture Handbook

Use this section when the task affects application structure, boundaries, data flow, auth architecture, API shape, or error strategy.

## Load Order

1. `architecture-overview.md`
2. the one or two task-specific docs below

## Suggested Usage

- API or server action structure: `api-architecture.md`
- auth flow or session boundary: `auth-architecture.md`
- backend layering: `backend-architecture.md`
- frontend composition and route-side structure: `frontend-architecture.md`
- cross-layer movement of data: `data-flow.md`
- failure/result strategy: `error-handling-architecture.md`

Do not load the full folder by default.
