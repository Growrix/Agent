# Component Library Checklist

A robust, token-driven component library for SaaS/product apps. Use this list to guide AI or human developers in building and maintaining a consistent, scalable UI system.

---

- Button (primary, secondary, icon, fab, ghost, link, status)
- Input (text, password, email, number, textarea)
- Select (dropdown, multi-select)
- Checkbox
- Radio
- Switch/Toggle
- Avatar
- Badge/Chip
- Tooltip
- Spinner/Loader
- Divider

## Layout & Structure
- Container
- Grid (responsive, 12-column)
- Stack (vertical/horizontal spacing)
- Card (with header, content, footer)
- Modal/Dialog
- Drawer/Sidebar
- Accordion/Collapse
- Tabs
- Stepper/Progress Bar
- Breadcrumbs
- Pagination

## Navigation
- App Bar/Top Navigation
- Side Navigation/Drawer
- Bottom Navigation (mobile)
- Menu (dropdown, context menu)
- Search Bar/Autocomplete

## Data Display
- Table (sortable, filterable, paginated)
- Data Grid (with selection, bulk actions)
- List (simple, with avatars, with actions)
- Empty State/Placeholder
- Statistic/Metric Card
- Chart/Graph (bar, line, pie, donut)
- Timeline/Activity Feed

## Feedback & Messaging
- Alert (success, error, warning, info)
- Toast/Snackbar
- Banner
- Dialog/Confirmation
- Tooltip
- Popover

## Forms & Validation
- Form (with validation, error summary)
- Fieldset/Legend
- File Upload/Dropzone
- Date Picker/Time Picker/Date Range Picker
- Slider/Range Input
- Tag Input/Chips Input

## Media & Visuals
- Image (responsive, lazy-loaded)
- Avatar Group
- Icon (token-driven, SVG)
- Carousel/Gallery
- Video Player (if needed)

- User Table (with roles, status, actions)
- Role/Permission Manager
- Audit Log/Activity Log
- Settings Panel (profile, preferences, notifications)
- Resource Management Table (CRUD)
- Bulk Actions Toolbar
- Filter/Sort Panel
- Status Indicator (active, pending, disabled, etc.)
- Status Button (toggle/quick status change)
- Rich Text Editor/Markdown Editor

## Public-Facing
- Hero Section
- Feature List/Grid
- Testimonial/Review Card
- Pricing Table
- FAQ/Accordion
- Blog Card/Blog List
- Newsletter Signup
- Footer (with links, social icons)
- Cookie Consent Banner

## Utility
- Skeleton Loader/Placeholder
- Scroll to Top Button
- Error Boundary/Fallback UI
- Theme Switcher (light/dark)

---

> Each component should be built using your design tokens and documented with props, variants, and usage guidelines. Expand this list as your app grows.
