# দ্রুত রেফারেন্স গাইড | Quick Reference Guide
## এজেন্টিক সিস্টেম ওভারভিউ (Agentic System Overview)

---

## 📌 এক নজরে সিস্টেম (System at a Glance)

### কী?
একটি পরিকল্পনা → বাস্তবায়ন সিস্টেম যা ওয়েবসাইট এবং অ্যাপ তৈরির প্রক্রিয়া স্বয়ংক্রিয় করে।

### কেন?
- ভুল কমায়
- সময় বাঁচায়
- সবাই একই পথে থাকে

### কারা?
- 7 টি এজেন্ট (বিশেষজ্ঞ)
- প্রতিটি আলাদা কাজ করে

---

## 🎯 ৭টি এজেন্ট এবং তাদের কাজ (7 Agents & Their Jobs)

| # | এজেন্ট | কাজ | আউটপুট |
|---|--------|------|---------|
| 1 | **Orchestrator** | সবার নিয়ম তৈরি করে | চুক্তি-পত্র |
| 2 | **Frontend Designer** | ডিজাইন এবং লেআউট | পেজ পরিকল্পনা |
| 3 | **Backend Planner** | সার্ভার কাজ | সার্ভার পরিকল্পনা |
| 4 | **API Architect** | ফ্রন্ট-ব্যাক সংযোগ | API স্পেক |
| 5 | **Security Expert** | নিরাপত্তা নিয়ম | নিরাপত্তা পরিকল্পনা |
| 6 | **DevOps Lead** | ডিপ্লয়মেন্ট | রিলিজ পরিকল্পনা |
| 7 | **QA Governor** | পরীক্ষা | টেস্ট প্ল্যান |

---

## 📊 প্রক্রিয়া প্রবাহ (Process Flow)

```
WEEK 1-2: পরিকল্পনা ফেজ
  ├─ Orchestrator → চুক্তি তৈরি
  ├─ Frontend → ডিজাইন পরিকল্পনা
  ├─ Backend → সিস্টেম পরিকল্পনা
  └─ API → যোগাযোগ নিয়ম

WEEK 3-4: নির্দিষ্টকরণ ফেজ
  ├─ Security → নিরাপত্তা যোগ করা
  ├─ DevOps → রিলিজ পরিকল্পনা
  └─ QA → টেস্ট কেস

WEEK 5+: বাস্তবায়ন ফেজ
  └─ ডেভেলপার → কোডিং শুরু
```

## 🎨 ফ্রন্টেন্ড ডেলিভারেবল (Frontend Deliverables)

### প্রতিটি পেজে এটি থাকে:

✅ **আর্কিটেকচার**
- সাইট ম্যাপ
- নেভিগেশন মডেল
- পেজ হায়ারার্কি

✅ **ডিজাইন**
- রঙ এবং ফন্ট
- লেআউট প্যাটার্ন
- অ্যানিমেশন গাইড

✅ **কম্পোনেন্ট**
- বাটন, কার্ড, ফর্ম
- ইন্টারঅ্যাকশন স্টেট
- মোবাইল অ্যাডাপ্টেশন

✅ **এন্টারপ্রাইজ**
- অ্যাক্সেসিবিলিটি (WCAG AA)
- মাল্টিল্যাঙ্গুয়াল সাপোর্ট
- পারফরম্যান্স (Lighthouse 90+)

## ✅ এন্টারপ্রাইজ চেকলিস্ট (Enterprise Checklist)

প্রতিটি পেজে এই বৈশিষ্ট্য আছে কি না যাচাই করুন:

### 1️⃣ অ্যাক্সেসিবিলিটি
- ☐ WCAG 2.1 AA সম্মত
- ☐ কীবোর্ড নেভিগেশন কাজ করে
- ☐ কালার কনট্রাস্ট ভালো

### 2️⃣ পারফরম্যান্স
- ☐ Lighthouse 90+ স্কোর
- ☐ লোড সময় < 2 সেকেন্ড
- ☐ ইমেজ অপটিমাইজড

### 3️⃣ মোবাইল
- ☐ মোবাইলে পরীক্ষিত
- ☐ টাচ বান্ধব
- ☐ রেসপন্সিভ লেআউট

### 4️⃣ নিরাপত্তা
- ☐ ইনপুট ভ্যালিডেশন
- ☐ XSS সুরক্ষা
- ☐ CSRF টোকেন

### 5️⃣ ডকুমেন্টেশন
- ☐ README লেখা
- ☐ ai-context.yaml আপডেটেড
- ☐ কম্পোনেন্ট ডকস লেখা

## 🔍 সাধারণ সমস্যা এবং সমাধান (Common Issues & Fixes)

### সমস্যা ১: ব্যাকএন্ড এখনও তৈরি হয়নি
**সমাধান:** মকড ডেটা দিয়ে ফ্রন্টেন্ড এগিয়ে যান, পরে সংযোগ করুন।

### সমস্যা ২: ডিজাইন সিস্টেম পরিবর্তিত হচ্ছে
**সমাধান:** মাস্টার ডিজাইন ফাইল লক করুন, কমিটির অনুমতি নিয়ে পরিবর্তন করুন।

### সমস্যা ৩: মোবাইলে ভাঙছে
**সমাধান:** রেসপন্সিভ ডিজাইন গাইড পুনরায় পড়ুন, পুনরায় পরীক্ষা করুন।

### সমস্যা ৪: অ্যাক্সেসিবিলিটি ব্যর্থ
**সমাধান:** WCAG গাইড পড়ুন, স্বয়ংক্রিয় টুল চালান, স্ক্রিন রিডার পরীক্ষা করুন।

## 🚀 দ্রুত শুরু (Quick Start)

### নতুন পেজ যোগ করার জন্য:

```
1. Frontend/home-page.md পড়ুন (উদাহরণ হিসেবে)
2. নতুন পেজের জন্য টেমপ্লেট তৈরি করুন
3. সমস্ত 13 বিভাগ যোগ করুন
4. মোবাইল এবং ডেস্কটপ উভয়ের জন্য নোট করুন
5. অ্যাক্সেসিবিলিটি নোট যোগ করুন
6. README তে পেজ যোগ করুন
```

### নতুন কম্পোনেন্ট যোগ করার জন্য:

```
1. 02-component-system.md তে যোগ করুন
2. ডিজাইন টোকেন ব্যবহার করুন
3. সব স্টেট ডকুমেন্ট করুন (ডিফল্ট, হোভার, ক্লিক)
4. মোবাইল সংস্করণ যোগ করুন
5. এক্সপোর্ট করুন এবং এক্সপোর্ট করুন
```
