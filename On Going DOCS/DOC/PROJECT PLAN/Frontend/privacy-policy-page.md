# Privacy Policy Page

## Page Definition
- Purpose: explain how visitor, chat, booking, and purchase data is handled.
- Target audience: all visitors, especially those using AI chat, forms, booking, and checkout.
- Primary CTA: Contact for Privacy Questions.
- Secondary CTA: Read Terms.

## Sections In Visual Order

### 1. Policy Header
- Content: effective date, summary statement, and quick links to key sections.
- Components: utility hero, anchor nav.

### 2. Data Collection Summary
- Content: what is collected through analytics, forms, AI chat, booking, and checkout.
- Components: content blocks, list cards.

### 3. Use of Data
- Content: service delivery, support, analytics, transaction processing, and product improvement.
- Components: content section.

### 4. Third-Party Services
- Content: Stripe, booking provider, analytics, hosting, chat, and WhatsApp usage notes.
- Components: table or structured content cards.

### 5. User Rights and Contact
- Content: access, correction, deletion, and privacy contact path.
- Components: content block, CTA row.

### 6. Final Legal Links
- Content: links to terms, contact, and support.
- Components: footer utility links.

## State Requirements
- Anchor nav should highlight the current section.
- Long-form content must remain readable and keyboard navigable.

## Responsive Adaptation
- Mobile uses a collapsible section index instead of a persistent side nav.

## SEO and Metadata
- Title: Privacy Policy.
- Description: Learn how the agency handles data related to contact forms, AI chat, booking, analytics, and payments.

## Conversion Path
- Policy review -> trust reinforcement -> contact if clarification is needed.
